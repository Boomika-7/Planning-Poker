# Planning Poker

A real-time Planning Poker web app for Agile teams to estimate stories collaboratively.

## Overview

This project is a React + TypeScript + Firebase application where users can:
- create a room,
- join with a 6-character room code,
- submit one vote from a Fibonacci deck,
- auto-reveal votes when everyone has voted,
- restart rounds,
- transfer host ownership on host exit,
- delete/exit sessions.

Users are authenticated with Firebase Anonymous Auth, and all room/participant updates are synchronized live using Firestore `onSnapshot` listeners.

## Tech Stack

- React 19
- TypeScript 5
- Vite 7
- React Router 7
- Firebase 12 (Auth + Firestore)
- Tailwind CSS
- Lucide React icons

## Features

- Anonymous sign-in on app load (`useAuthUser`)
- Create room with autogenerated unique 6-char ID (`createSession`)
- Join existing room by ID (`joinSession`)
- Real-time participant list and room state (`useParticipants`, `useSession`)
- Vote once per round (vote lock after submit)
- Automatic vote reveal when all participants have voted (`setVote`)
- Host-only controls:
  - restart round (`restartSession`)
  - delete room (`deleteSession`)
- Exit behavior (`exitSession`):
  - non-host leaves room
  - host transfer to earliest joined remaining participant
  - delete room if host was last participant
- Invite link copy from room page
- Loading screens during auth/session validation

## Routes

- `/` -> Home page (Create/Join UI)
- `/session/:sessionId` -> Session room

## Firestore Data Model

### Collection: `sessions`

Document ID: `sessionId` (6-char uppercase string)

Fields:
- `name: string`
- `hostId: string`
- `hostName: string`
- `revealVotes: boolean`
- `createdAt: Timestamp`

### Subcollection: `sessions/{sessionId}/participants`

Document ID: `uid`

Fields:
- `name: string`
- `isHost: boolean`
- `vote: string | null`
- `joinedAt: Timestamp`
